# A01 GameSpec 基础

状态：已完成

## 目标

新增不依赖 AI、UI、流水线和具体引擎的 `adm-new-game-spec` crate，定义 V2 规范的最小类型核心。

## 变更

- 定义身份、意图、能力轴、实体/组件/关系、动作、状态机、资源、空间、时间、交互、内容、表现、技术约束、验收场景、追踪和扩展类型。
- 为 ID 提供受验证的新类型；无序集合使用稳定键顺序，有序效果、迁移和分支保持输入顺序。
- 未知扩展必须进入命名空间块，不能通过散落的 `flatten Value` 污染核心。
- 添加通道防守、三消、分支叙事、回合战术四个 JSON 样例。
- 添加反过拟合词汇门禁和 JSON 往返测试。

## 验收

- 四个样例使用同一 schema 反序列化并完成语义无损往返；不要求保留 JSON 空白或对象键原始顺序。
- 核心字段不包含品牌、Unity、塔防、卡牌或叙事专属名称。
- 新 crate 单测、工作区检查和安全自检通过。

## 回滚

删除新 crate 和 workspace 成员即可；不触及现有运行路径和保存格式。

## 实际结果

- 新增 `adm-new-game-spec`，当前没有应用层依赖，也未接入现有运行路径。
- 建立严格 camelCase JSON 契约、受验证 `SpecId`、带类型的跨对象引用和命名空间扩展。
- 新增通道防守、三消网格、分支叙事、回合战术四个样例。
- 新增 6 项测试，覆盖 ID、应用层依赖隔离、核心词汇反过拟合、严格未知字段、跨结构差异和语义往返。
- `cargo fmt --all -- --check`、`cargo check --workspace --locked`、`cargo test --workspace --locked` 和安全自检均通过。

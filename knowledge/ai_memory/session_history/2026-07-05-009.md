## 摘要

继续 goal 模式推进 Rust 全项目重建，完成 pipeline resume/失败恢复的底层能力，并让应用层默认 pipeline 使用 state-aware runner 直接维护 `PipelineRunState`。

## 完成内容

- `adm-pipeline`
  - `PipelineRunState::complete_stage(...)` 现在避免重复记录同一阶段。
  - 新增 `PipelineRunState::cancel(...)`。
  - 新增 `PipelineRunState::is_stage_completed(...)`。
  - 新增 `PipelineRunState::validate_for_graph(...)`：
    - 拒绝未知 completed stage。
    - 拒绝未知 active stage。
    - 拒绝 completed stage 缺少已完成依赖的坏状态。
  - 新增 `PipelineRunner::run_serial_with_state(...)`：
    - 跳过已完成阶段。
    - 执行剩余阶段。
    - 成功时更新 completed stages。
    - 失败时写入 Failed run state 并停止。
    - 全部完成后写入 Succeeded run state。
- `adm-application`
  - 默认核心 pipeline 改为使用 `run_serial_with_state(...)`。
  - `pipeline/run_state.adm` 现在由 runner 执行时直接维护，而不是从 report 反推。
  - 应用层测试新增从正式存档读取并解析 `content/pipeline/run_state.adm` 的断言。

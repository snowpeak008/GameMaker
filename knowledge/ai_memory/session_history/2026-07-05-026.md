# 会话记录：2026-07-05-026

## 摘要

继续 goal 模式推进 Rust 全项目重建，在 `adm-application` 增加从配置构建真实 chat-completions remote provider 的便捷入口，并在 `adm-cli` 增加无网络 dry-run 检查命令。该命令只验证 provider 配置、secret 解析、模型名和 capability 支持，不触发真实 AI 网络请求。

## 完成内容

- `adm-application`
  - 新增 `ConfiguredChatCompletionsProvider` type alias。
  - 新增 `chat_completions_provider_from_config(provider_id, model)`。
    - 创建 `ReqwestBlockingHttpJsonClient`。
    - 创建 `ChatCompletionsTransport`。
    - 复用 `remote_ai_provider_from_config(...)`。
    - 从配置读取 endpoint、secret_ref 和 capabilities。
  - 新增无网络单元测试：
    - 使用 `env:PATH` 作为可解析 secret。
    - 只构建 provider 并检查 capability 支持。
    - 不调用 provider `run()`，避免真实网络请求。
- `adm-cli`
  - 新增 `ai-provider-check <provider_id> <model> [data_root]`。
  - 输出：
    - `provider_id`
    - `model`
    - `network_call=false`
    - 所有已知 capability 的支持矩阵。
  - `ai-provider-check` 通过 `chat_completions_provider_from_config(...)` 验证配置和 secret 是否可用。

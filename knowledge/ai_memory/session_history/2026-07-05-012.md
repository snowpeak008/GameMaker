# 会话记录：2026-07-05-012

## 摘要

继续 goal 模式推进 Rust 全项目重建，完成应用层 pipeline executor 从占位产物到真实 artifact registry 产物的切换，并增加 AI provider failure kind 分类与持久化。

## 完成内容

- `adm-application`
  - 默认 pipeline 不再使用 `artifact_{stage_id}` 占位产物。
  - 新增 `CoreStageExecutor`，执行阶段时从 `ArtifactRegistry` 查询当前 stage 的真实 artifact。
  - stage 无注册 artifact 时返回 Failed。
  - `run_default_pipeline(...)` 改为接收 `ArtifactRegistry`，并通过 state-aware runner 生成 report/state。
  - 应用层测试新增断言：pipeline report 中包含真实 `artifact_package_manifest`。
- `adm-ai`
  - 新增 `AiFailureKind`：
    - `BudgetExceeded`
    - `UnsupportedCapability`
    - `ProviderUnavailable`
    - `InvalidResponse`
    - `ValidationRejected`
    - `Unknown`
  - `AiTaskRecord` 新增 `failure_kind`。
  - `AiTaskRecord::fail_with_kind(...)` 新增。
  - `ADM_AI_JOURNAL_V1` 新增 `failure_kind=` 字段，旧格式缺省可解析为 `None`。
  - `AiProviderRouter` 现在分类：
    - 预算不足 -> `BudgetExceeded`
    - 无支持 provider -> `UnsupportedCapability`
    - provider validation/invalid input -> `InvalidResponse`
    - provider IO/conflict -> `ProviderUnavailable`
  - 新增失败分类测试。

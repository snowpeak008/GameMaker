## 摘要

继续 goal 模式推进 Rust 全项目重建，完成 AI task journal 的稳定摘要层，并将 AI 失败数量、失败分类和最近错误暴露到桌面 GUI 与 CLI。

## 完成内容

- `adm-ai`
  - 新增 `AiFailureSummary`。
  - 新增 `AiTaskJournalSummary`。
  - 新增 `AiTaskJournal::summary()`。
  - summary 统计：
    - record count
    - accepted count
    - failed count
    - rejected count
    - failure kind counts
    - last failure kind
    - last error
  - `AiFailureKind` 增加排序能力，用于稳定输出 failure summary。
- `adm-ui-model`
  - `AiStatusView` 增加 `failed_count`、`rejected_count`、`failure_summary`、`last_error`。
  - `AiStatusView::render()` 输出失败/拒绝数量，并在有失败时输出失败分类与最近错误。
  - 新增 UI model 单元测试覆盖失败信息渲染。
- `adm-desktop`
  - 桌面 run/resume summary 使用 `AiTaskJournal::summary()`。
  - 项目检查中的 AI 面板使用同一套 summary helper。
  - smoke 输出现在包含 `failed=0`、`rejected=0`，未来出现 provider 失败时会显示 failure summary 与 last error。
- `adm-cli`
  - 新增 `ai-journal <archive_id> [data_root]`。
  - CLI 可直接读取正式存档中的 `content/ai/journal.adm` 并输出 AI 摘要。

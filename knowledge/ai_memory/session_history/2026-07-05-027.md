# 会话记录：2026-07-05-027

## 摘要

继续 goal 模式推进 Rust 全项目重建，完善 Slint 桌面 POC 的 AI provider 配置体验：capabilities 不再固定为默认值，改为可编辑输入；桌面增加 provider Check 按钮，执行与 CLI 相同原则的无网络 dry-run 配置检查。

## 完成内容

- `adm-desktop` UI
  - 新增 `ai-provider-capabilities` 输入，默认 `text_generation`。
  - 新增 `ai-provider-model` 输入，默认 `gpt-test`。
  - 新增 `check-ai-provider` callback。
  - Provider 配置区拆为两行，避免一行过长：
    - Provider id + endpoint。
    - secret_ref + capabilities + model + Save/Check/Disable。
- `adm-desktop` Rust 逻辑
  - 保存 provider 时解析 capabilities 输入。
  - 空 capabilities 输入默认补 `text_generation`。
  - 支持逗号分隔 capability 列表。
  - 使用 `AiProviderConfig::validate()` 继续拦截未知、空、重复能力等错误。
  - 新增 `check_ai_provider_config(...)`：
    - 构建 chat-completions provider。
    - 验证配置、secret、model。
    - 输出 `network_call=false`。
    - 输出所有已知 capability 支持矩阵。
    - 不调用真实 AI 网络。
- smoke 覆盖
  - 保存 `remote_ready` provider，capabilities 为 `text_generation,structured_output`。
  - 使用 `env:PATH` 验证 secret 可解析。
  - 调用 provider check dry-run。
  - 验证输出包含 `network_call=false` 和 `supports.structured_output=true`。
  - 最终 AI diagnostics 显示 `remote_ready | Ready | text_generation,structured_output`。

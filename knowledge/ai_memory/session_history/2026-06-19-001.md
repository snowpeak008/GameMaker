## 关键发现

1. **架构洞察**
   - `core/engines/generation.py` 是全项目最重的文件（3795行），16个阶段逻辑全在里面
   - 每个步骤插件的模式完全统一：stage_id + _source_groups + execute()
   - 制品依赖通过 `artifact_layer/registry.json` 声明，拓扑排序决定执行顺序

2. **配置发现**
   - 默认 API 配置指向代理：`https://vip.auto-code.net/v1`，模型 `gpt-5.5`
   - 步骤03+ 依赖 Unity 配置（`settings/project_settings.json`），未配置时产出 blocked 状态
   - `ucos/` 目录的大部分文件已被删除（git status 显示为 D），只保留 episodic 记忆

3. **技术债务**
   - GUI 卡顿问题严重：render() 全量重建 + 规则引擎每次全量计算
   - `generation.py` 过大，但16个 _stageN_outputs() 互不调用，可拆但优先级不高
   - `CommercialDesignApp` 当前继承 tk.Tk，计划改为 tk.Frame 嵌入主框架

4. **命名约定**
   - 步骤目录：`step_{NN}_{slug}/`（如 `step_03_program_requirements/`）
   - 适配器文件：`{model}_adapter.py`
   - 引擎文件：功能名词（`generation.py`, `delta_patch.py`）

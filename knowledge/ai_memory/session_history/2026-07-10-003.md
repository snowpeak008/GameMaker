# Session 2026-07-10-003

## Summary

Compared the Python and NEWrust save systems from UI through storage, then completed
the NEWrust/Tauri save manager as a transaction-safe, multi-window, recoverable usable
version. The final portable build was tested with a real clean exit and restart, and is
left running for user trial.

## Python / Rust Findings

- Python's 980x520 Treeview offers new/save-to-selected/load/rename/delete/open/refresh,
  but filters out archives without a design object and relies mainly on PID metadata
  locks and multi-file mutations without a durable transaction journal.
- Python's save-to-selected path can combine one project's design with another save's
  pipeline workspace, so NEWrust intentionally does not reproduce it.
- Useful Python behavior was retained: session drafts, formal archives, deleted-current
  detached copies, file maps, full/delta snapshots, timeline history, blank-save cleanup,
  and verified `design_project` execution-object recovery.
- NEWrust now separates New Project Save, Save As Copy, and Save Current, and requires an
  explicit save/discard/cancel choice before switching saves.

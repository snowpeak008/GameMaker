# 会话记录：2026-07-05-025

## 摘要

继续 goal 模式推进 Rust 全项目重建，在 `adm-ai` 中实现真实 HTTP JSON client adapter。该 adapter 基于 `reqwest` blocking client，面向 chat-completions transport 的真实 provider 调用；测试仍保持无真实网络依赖，只覆盖本地输入校验和 secret 不泄露。

## 完成内容

- `adm-ai`
  - 新增 `reqwest` 直接依赖：
    - `default-features = false`
    - features: `blocking`, `json`, `rustls`
  - 新增 `ReqwestBlockingHttpJsonClient`。
    - 内部使用 `reqwest::blocking::Client`。
    - 默认请求超时 60 秒。
    - 实现 `AiHttpJsonClient`。
  - `post_json(...)` 行为：
    - 拒绝空 endpoint。
    - 拒绝空 bearer secret。
    - 拒绝空 request body。
    - 用 bearer auth 发送 JSON POST。
    - 设置 `Content-Type: application/json`。
    - 设置 `Accept: application/json`。
    - 非 2xx HTTP status 返回 `AdmErrorKind::Io`。
    - HTTP error body 截断到 512 字符，避免错误信息过长。
    - 不把 bearer secret 写入错误信息。
- 单元测试
  - `reqwest_http_client_rejects_invalid_inputs_without_network`。
  - `reqwest_http_client_does_not_include_secret_in_url_parse_error`。

# 会话记录：2026-07-05-033

## 摘要

继续 goal 模式推进 Rust 全项目重建，新增真实 AI 调用的显式手动入口。默认 smoke、检查和普通配置路径仍不联网，只有用户主动点击桌面 `Invoke` 或执行 CLI `ai-provider-invoke` 时才会触发配置供应商的网络调用。

## 完成内容

- `adm-cli`
  - 新增 `ai-provider-invoke <provider_id> <model> <prompt> [data_root]` 命令。
  - 命令通过已保存的 provider profile 构建 chat-completions provider。
  - 命令创建 `AiTaskRequest` 并调用 `provider.run(...)`，这是显式真实网络调用入口。
  - 输出包含 `provider_id`、`model`、`network_call=true`、`output_state`、`raw_output` 和可选 `validation_notes`。
  - 参数不足时返回 `InvalidInput`，不触发网络调用。
- `adm-desktop`
  - 新增 Slint `ai-provider-prompt` 输入。
  - 新增 `invoke-ai-provider()` callback。
  - AI Config 面板新增 `Invoke` 按钮。
  - 新增 `invoke_ai_provider_config(...)`，使用当前 provider/model/prompt 主动调用真实 provider。
  - 新增 `compact_status_output(...)`，限制状态栏输出长度，避免长模型响应撑坏界面。
  - smoke 不点击 `Invoke`，保持默认无网络。

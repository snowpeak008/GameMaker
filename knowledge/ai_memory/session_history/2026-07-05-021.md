# 会话记录：2026-07-05-021

## 摘要

继续 goal 模式推进 Rust 全项目重建，在 `adm-application` 中增加从 provider 配置到运行期远程 provider 的工厂边界。该边界用 `AiProviderConfig + EnvSecretResolver + AiRemoteTransport` 构建 `RemoteAiProvider`，仍然只在运行期解析 secret，不把 API key 写入配置文件。

## 完成内容

- `adm-application`
  - 新增 `AdmApplication::remote_ai_provider_from_config(...)`。
  - 输入：
    - provider id
    - capabilities
    - transport
  - 行为：
    - 查找 provider 配置。
    - 拒绝未配置 provider。
    - 拒绝 disabled provider。
    - 要求 endpoint hint。
    - 要求 secret ref。
    - 使用 `EnvSecretResolver` 运行期解析 secret。
    - 使用 `AiSecretMaterial` 包装 secret。
    - 返回 `RemoteAiProvider<T>`。
- 单元测试
  - 新增 `ApplicationRemoteTransport` fake transport。
  - 新增 `application_builds_runtime_remote_provider_from_config`。
  - 使用 `env:PATH` 作为可解析 secret ref。
  - 验证 fake transport 收到 endpoint 和非空 secret。
  - 验证 profile 中只保存 `env:PATH`，不保存环境变量真实值。

# 会话记录：2026-07-05-037

## 摘要

继续 goal 模式推进 Rust 全项目重建，新增 Slint desktop 的显式陈旧锁清理入口。该功能不会自动删除未知锁，只在用户点击 `Clear Lock` 时对当前选中 archive 执行清理，用于处理强制关闭后遗留 `.archive_lock` 导致后续加载失败的情况。

## 完成内容

- `adm-desktop`
  - Slint 增加 `clear-selected-lock()` callback。
  - Archive 操作行增加 `Clear Lock` 按钮。
  - 新增 `clear_selected_archive_lock_file(...)`。
  - 如果当前窗口持有选中锁，`Clear Lock` 会释放当前窗口锁。
  - 如果是外部/陈旧锁，`Clear Lock` 会读取 owner 信息并删除当前 archive 根目录下 `.archive_lock`。
  - 如果没有锁，返回明确的无锁提示。
  - 清锁后刷新项目列表和详情。
  - desktop smoke 覆盖外部锁清理后 `locked=false` 的恢复路径。
- 发布产物
  - 重新构建 release。
  - 重新 staging 到 `RUST/dist/AutoDesignMaker-rust/AutoDesignMaker-rust.exe`。

# 2026-06-21-003 会话记录

## 摘要

按 `plan/adr/0001-sandbox-replaced-by-per-session-drafts.md` 和 `plan/adr/0002-formal-archive-excludes-snapshots.md` 完成路径与存档结构调整。

## 完成内容

- `core/paths.py` 新增 per-session draft 路径：`drafts/{timestamp}_{pid}/`，旧 `SANDBOX_DIR` 保留为兼容别名但实际指向当前 draft。
- `core/save/manager.py` 改为从当前 draft 同步 active 文件，正式存档目录 `saves/{save_id}/` 只保留 `manifest.json` 和 `workspace/`。
- 正式存档不再写 `snapshots/`、`save_file_map.json`、`timeline.jsonl`；这些事务记录只写入当前 draft。
- 新增 `draft_meta.json` 写入，记录 session、pid、draft 根和关联正式存档路径。
- 修复旧 `save_manifest.json` 兼容读取，并在同步时迁移为 `manifest.json`。
- 将 artifact preflight、runtime control、design runtime root、ucos bridge 等旧路径推导切换到 draft/当前路径体系。
- `.gitignore` 新增 `drafts/`，避免运行时草稿进入提交。
- 更新旧测试导入路径并补充 draft/archive 行为测试。

## 自查发现并修复

- 修复 `ai_ucos_bridge` 在 runtime root 改为 draft 后可能把 ucos 写到 draft 下的问题，改为优先使用 `core.paths.UCOS_DIR`。
- 修复 `data_loader.source_project_root()` fallback 指向 `core/` 的边缘问题。
- 修复 `clear_active_workspace()` 未清理 draft runtime history（`snapshots/`、`draft_file_map.json`、`timeline.jsonl`）的问题。
- 修复旧测试仍引用已删除 `src.*` 包路径的问题。

# 会话记录：2026-07-05-014

## 摘要

继续 goal 模式推进 Rust 全项目重建，将 `adm-application` 的核心流水线从“先生成全部文档、再用 registry 报成功”的占位式执行，拆成按 stage 执行业务输出的真实 stage executor。

## 完成内容

- `adm-application`
  - 新增 `src/core_pipeline.rs`，把核心流水线业务实现从 `lib.rs` 拆出。
  - `CorePipelineServices::build(...)` 现在创建 pipeline graph，并通过 `PipelineRunner::run_serial_with_state(...)` 驱动业务 executor。
  - 新增 `CorePipelineStageExecutor`：
    - `design` stage 生成设计文档、执行 AI intervention journal、登记 `artifact_design_project`。
    - `development` stage 生成开发计划、登记 `artifact_development_plan`。
    - `assets` stage 生成资源计划、登记 `artifact_asset_plan`。
    - `sdk` stage 生成 SDK 知识索引、登记 `artifact_sdk_index`。
    - `packaging` stage 校验输入、生成 package manifest、登记 `artifact_package_manifest`。
  - `CorePipelineAssembly` 保存每个 stage 的中间业务对象和输出文档，最终汇总 validation、pipeline report、run state、artifact registry 和 AI journal。
  - `lib.rs` 删除旧的 `CoreStageExecutor` 和旧的预生成输出逻辑，只保留应用服务边界。
- 测试
  - 新增 `core_pipeline_builds_outputs_from_stage_executor`，验证 stage executor 生成 5 个 stage result、5 个 artifact、packaging artifact 和通过状态。

# 会话记录：2026-07-05-007

## 摘要

继续 goal 模式推进 Rust 全项目重建，完成 AI 配置/密钥解析/运行诊断基础，并把 AI 干预策略、预算和重试策略接入核心流水线。

## 完成内容

- `adm-config`
  - `AppConfig` 新增 `ai: AiSettings`。
  - 新增 `config/app_config.adm` 的保存、加载和默认配置保留。
  - 新增 `AiProviderConfig`：
    - enabled/disabled/local provider 形态
    - display name
    - endpoint hint
    - `requires_secret`
    - `secret_ref`
  - 新增 `SecretRef`：
    - `env:NAME`
    - `named:NAME`
  - 新增 `EnvSecretResolver`。
  - 新增 provider readiness 诊断：
    - Disabled
    - Ready
    - MissingSecret
    - UnsupportedSecretReference
  - 新增 AI 预算、重试、干预策略配置：
    - `default_budget_units`
    - `retry.max_attempts`
    - `intervention.min_quality_score_percent`
    - `intervention.on_quality_gap`
    - `intervention.on_missing_content`
    - `intervention.review_after_generation`
- `adm-ai`
  - 新增 `AiInterventionCriteria`。
  - 新增 `AiInterventionDecision` 与原因模型：
    - quality score below threshold
    - missing required content
    - post-generation review
  - 新增 `decide_ai_intervention(...)`。
  - 新增 `AiRetryPolicy`。
  - `AiTaskRecord` 新增 `attempts`。
  - `AiProviderRouter` 新增 `run_with_budget_and_policy(...)`，支持 provider 失败重试。
- `adm-application`
  - `AdmApplication::for_data_root(...)` 改为优先加载已有配置。
  - 应用启动时确保 `config/app_config.adm` 存在。
  - 新增 `save_config()`。
  - 新增 `ai_diagnostics()` 和 `AiDiagnosticsReport`。
  - 核心流水线改为使用 `AiSettings` 决定是否触发 AI：
    - 分数不足
    - 内容缺失
    - 生成后审查
  - AI 运行使用配置中的预算和重试次数。
- `adm-cli`
  - 新增 `ai-doctor [data_root]` 命令，输出 AI provider readiness、预算和重试设置。

# 会话记录：2026-07-05-024

## 摘要

继续 goal 模式推进 Rust 全项目重建，将 AI provider capabilities 纳入 `adm-config` profile schema，并让 `adm-application` 的 remote provider factory 从配置读取能力列表，不再由调用方绕过配置传入。CLI 和 Slint POC 保存 provider 时默认写入 `text_generation`，诊断面板也显示 provider capabilities。

## 完成内容

- `adm-config`
  - 新增 `adm-ai` 依赖，用 `AiCapability` 作为配置字段类型。
  - `AiProviderConfig` 新增 `capabilities: Vec<AiCapability>`。
  - enabled/local provider 默认能力为 `text_generation`。
  - disabled provider 默认能力为空。
  - profile 渲染新增 `ai.provider.N.capabilities=...`。
  - profile 解析支持逗号分隔 capabilities。
  - 旧 profile 缺少 capabilities 时：
    - enabled provider 默认补 `text_generation`。
    - disabled provider 保持空能力。
  - provider 校验拒绝 enabled provider 空能力和重复能力。
  - provider diagnostics 输出 capabilities。
- `adm-application`
  - `remote_ai_provider_from_config(...)` 改为使用 `provider.capabilities.clone()`。
  - 运行时 provider 构建不再需要调用方单独传 capabilities。
- `adm-cli`
  - `ai-provider-set` 保持旧参数兼容，保存 provider 时写入默认 `text_generation` capability。
- `adm-desktop` / `adm-ui-model`
  - 保存 provider 时写入默认 `text_generation` capability。
  - AI diagnostics UI 文本显示 provider capabilities。
- 单元测试
  - `adm-config` 增加 provider capabilities profile round-trip 测试。
  - 更新配置 round-trip、diagnostics、application runtime provider factory、UI diagnostics 测试。

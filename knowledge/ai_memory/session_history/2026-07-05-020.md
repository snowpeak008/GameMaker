# 会话记录：2026-07-05-020

## 摘要

继续 goal 模式推进 Rust 全项目重建，在 `adm-ai` 中加入真实 AI provider adapter 的最小安全抽象：远程 provider、运行期 secret material、可替换 transport 和不泄露密钥的 debug 表示。当前不接真实网络，不写入 API key，只为后续供应商协议实现建立稳定边界。

## 完成内容

- `adm-ai`
  - 新增 `AiSecretMaterial`。
    - 运行期携带已解析 secret。
    - 构造时拒绝空 secret。
    - `Debug` 固定输出 `REDACTED`，避免日志泄露。
  - 新增 `AiRemoteRequest`。
    - 包含 provider id、endpoint hint、secret material、capability、prompt、context summary。
    - 提供只读 accessor。
    - `Debug` 通过 `AiSecretMaterial` 自动脱敏。
  - 新增 `AiRemoteResponse`。
  - 新增 `AiRemoteTransport` trait。
    - 后续真实 HTTP/OpenAI/Claude adapter 可实现该 trait。
    - 单元测试使用 fake transport，不打真实网络。
  - 新增 `RemoteAiProvider<T: AiRemoteTransport>`。
    - 实现现有 `AiProvider` trait。
    - 校验 endpoint hint、capabilities 和 capability support。
    - 将 `AiTaskRequest` 转换为 `AiRemoteRequest`，由 transport 发送并返回 `AiTaskResult`。
- 单元测试
  - 覆盖 remote provider 调用 transport。
  - 覆盖 debug 文本不包含真实 secret。
  - 覆盖空 secret 和空 endpoint 拒绝。

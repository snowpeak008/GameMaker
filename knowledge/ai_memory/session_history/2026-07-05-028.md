# 会话记录：2026-07-05-028

## 摘要

继续 goal 模式推进 Rust 全项目重建，将 Slint 桌面的 stage progress 从固定的 5 个按钮升级为真正的 `stage-items` 列表模型。Rust 层继续保留原有文本渲染用于 smoke/日志，同时将 `Vec<StageProgressItem>` 转换为 Slint `ModelRc<VecModel<StageRow>>`，UI 可重复渲染每个 stage 行并逐行 Inspect。

## 完成内容

- `adm-desktop` Slint UI
  - 新增 `StageRow` struct：
    - `id`
    - `label`
    - `status`
    - `artifacts`
    - `message`
  - 新增 `stage-items: [StageRow]` property。
  - 删除固定 `Design / Development / Assets / SDK / Packaging` 按钮组。
  - 新增可重复渲染的 Stages 面板：
    - 每行展示 label、status、artifact count、message。
    - 每行有 `Inspect` 按钮。
  - 新增统一 `inspect-stage(string)` callback。
- `adm-desktop` Rust
  - `DesktopRunSummary` 和 `ProjectInspection` 新增 `stage_items: Vec<StageProgressItem>`。
  - 新增 `stage_progress_items_from_runtime(...)`。
  - 新增 `stage_progress_items_from_parts(...)`。
  - `load_project_inspection(...)` 从已保存 run_state/run_report/artifact_registry 构建 stage item 列表。
  - `apply_run_summary(...)` / `apply_project_inspection(...)` 同步设置 Slint `stage-items` model。
  - 新增 `apply_stage_items(...)` 和 `stage_row_from_item(...)`。
  - 旧固定 stage callback 改为统一 `on_inspect_stage(...)`。
  - 删除不再使用的旧文本 helper，保持 warning-free。
- smoke 覆盖
  - resume 后断言 `ui.get_stage_items().row_count() == 5`。
  - 继续验证 `Packaging: Completed` 和 `package/manifest.adm` stage detail。

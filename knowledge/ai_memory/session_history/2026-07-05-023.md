# 会话记录：2026-07-05-023

## 摘要

继续 goal 模式推进 Rust 全项目重建，在 `adm-config` 中给应用配置 profile 增加显式 schema version，并保持旧 profile 的兼容读取。该改动为后续 provider capabilities、AI 配置迁移、默认值升级和正式发布前配置迁移打基础。

## 完成内容

- `adm-config`
  - 新增 `APP_CONFIG_PROFILE_VERSION: u32 = 1`。
  - `AppConfig` 新增 `profile_version` 字段。
  - `AppConfig::for_data_root(...)` 默认写入当前 profile version。
  - `AppConfig::validate()` 新增版本校验：
    - 拒绝 `profile_version == 0`。
    - 拒绝高于当前程序支持值的未来版本。
  - `render_profile()` 在 profile 文件头部写入 `profile_version=1`。
  - `from_profile_text()` 支持读取 `profile_version`。
  - 旧 profile 若缺少 `profile_version`，按当前版本兼容读取。
- 单元测试
  - `app_config_round_trips_profile_text`：验证 profile version 可往返并被渲染。
  - `app_config_accepts_legacy_profile_without_version`：验证旧 profile 兼容。
  - `app_config_rejects_future_profile_version`：验证未来版本拒绝。

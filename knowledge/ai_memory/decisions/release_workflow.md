# 仓库同步与验收流程

## 当前决策（2026-07-15）

- 远端验收基线为 `origin/main` 的 `a29c7d0`，标签 `v0.1.0` 保持不变。
- 从该决策生效起，可以在本地更新项目记忆，但验收完成前不得执行 `git commit`、`git tag`、`git push`、创建 GitHub Release 或其他远端修改。
- 记忆更新保持为本地未提交差异，不代表已经进入待发布版本。
- 只有用户明确说明验收通过并要求同步仓库后，才可以开始仓库同步流程。

## 验收后同步门禁

1. 先检查并汇报本地全部待提交差异，区分记忆、代码、计划和生成物。
2. 重新运行与改动范围相匹配的测试、独立性检查和安全扫描。
3. 向用户报告拟提交内容、验证结果和版本号。
4. 获得明确同步指令后，才能提交、打标签或推送。
